Rules

Everything you need to know to play.

Overview

This is an Attack/Defense CTF. Each team runs the same set of services on their own machine. You score points by keeping your services up (SLA bonus) and by stealing flags from other teams. Flags stolen from you cost you points. The team with the most points at the end wins.

Your API Key

You will receive an API key from the admin before the game starts. Keep it secret - it identifies your team for all API calls. Pass it in every request via the X-Team-Key header.

Getting Flag IDs

Flag IDs tell you where to look for flags on other teams' machines. Fetch them at the start of each tick.

GET /api/flagids/<service_id>

Response:

{ "10.10.1.2": ["user_42", "user_43"], "10.10.1.3": ["user_17"] }

Each key is a team's IP address. Each value is the list of flag IDs for that team on that service. Use the flag IDs to retrieve the actual flag from the target's service. Only the last 5 ticks of flag IDs are available.

Submitting Flags

Submit captured flags via a POST request. You can submit one flag at a time or a list of flags in a single request (maximum 50 per request).

POST /api/submit
X-Team-Key: <your_api_key>
Content-Type: application/json

# Single flag format:
{"flag": "NPW_exampleflag123"}

# Bulk flag format:
{"flags": ["NPW_flag1", "NPW_flag2"]}

API Response status codes:

ACCEPTED Flag valid. Points awarded.
INVALID Flag does not exist.
OWN_FLAG Flag belongs to your team (no points).
EXPIRED Flag older than 5 ticks.
ALREADY_CAPTURED Flag already claimed by someone else.

Rate limit: 1 request per second per team. Submitting excessive invalid flags may result in restrictions.

Scoring

SLA bonus (per tick) - for each service that is UP at the end of a tick, your team earns K points. Keeping your services running is rewarded every tick.
Attack points - when you capture a flag, you earn 10 * your_SLA points. The better your own services are running, the more your attacks are worth.
Defense penalty - when a flag is stolen from you, you lose 5 * your_SLA points. If your service is already down, the penalty is reduced.

SLA

SLA (Service Level Agreement) is calculated as the percentage of ticks in which your service was fully operational. A service is considered UP only if all three checker phases pass: check, put, and get. SLA affects both your attack points and your defense penalty, so keeping your services stable is always beneficial.
